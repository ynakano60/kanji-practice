# 漢字テスト練習アプリ 要件定義書 v1

作成日: 2026-07-12

## 1. 目的

小学校で配布される漢字テストの範囲プリント(例:「六月の漢字テスト範囲②」)に対応した書き取り練習を、スマホで手軽にできるようにする。プリントを撮影するだけで問題が登録され、子どもが一人で反復練習できることを目指す。

## 2. 利用者と役割

| 役割 | 利用者 | 主な操作 |
|------|--------|----------|
| 管理者 | 親 | プリント撮影・問題登録、問題の確認・修正、成績の確認 |
| 学習者 | 子ども(小学生) | 問題を解く、答え合わせ(自己採点)、間違い直し |

まずは自分の家族で使うが、最終的には URL を共有して他の家庭でも幅広く使えるようにすることを目指す。そのため、API キーがなくてもアプリの主要機能(手入力取り込み・OCR 貼り付け取り込み・練習・履歴)がすべて使える設計とする。

**データは家庭(端末)ごとに完全に独立**とする。共有サーバーやデータベースは持たず、各家庭が登録した問題・履歴はその端末内にのみ保存され、他の利用者からは見えない。同一家庭内で端末をまたぐ場合はバックアップ機能(F-15)のエクスポート/インポートでデータを移す。

## 3. 教材(入力データ)の構造

学校のプリントの形式に合わせる:

- 1問 = 「問題文(ひらがな読み+文脈語)」と「正解の漢字表記」のペア
  - 例: 問題「**しきん**を集める」 → 正解「資金」
  - 下線部(漢字に直す部分)と、そのまま残る部分(送りがな・文脈)がある
  - 送りがな付きの訓読みも多い(例: こころよい → 快い、けわしい道 → 険しい道)
- 問題セット = テスト1回分の範囲(例: 「6月範囲②」= 41〜80番の40問)
- 各問題に番号が振られている

## 4. 機能要件

### 4.1 問題登録(親向け)

問題の取り込みはすべてスマホの Web アプリ内で完結する。方式は次の3種類で、いずれも最後は共通の確認・修正画面(F-2)を通ってから保存する:

- **F-1 写真取り込み(AI 自動抽出)**: プリントをスマホカメラで撮影またはアルバムから選択し、AI で問題文・正解・番号を自動抽出する。エンジンは **Google Gemini(無料枠あり・推奨)** と **Anthropic Claude(有料)** の2種に対応し、設定済みのキーに応じて選択できる。API はブラウザから直接呼び出す(F-13 の API キーを使用)。キー未設定の場合はこのメニューを非表示にせず、F-16 の貼り付け取り込みを案内する
- **F-16 OCR 貼り付け取り込み(キー不要)**: スマホ標準の OCR 機能(iPhone のテキスト認識表示 / Google レンズ)で写真からコピーしたテキストをアプリに貼り付けると、アプリが「番号・問題文・正解」のペアに自動整形する。API キーなしで写真からの取り込みに近い体験を提供する
- **F-2 取り込み結果の確認・修正**: 抽出・整形結果を一覧表示し、誤認識を手動で修正・追加・削除できる。確定してから問題セットとして保存する
- **F-3 手入力取り込み**: 写真なしでも1問ずつ手入力で追加できる(単独の登録手段としても、取り込み失敗時の補完としても使う)
- **F-4 問題セット管理**: 取り込んだ問題セットには任意の表題(例「5年生の6月漢字テスト用」)を付けて保存する。カテゴリ(タグ)による分類にも対応し、一覧からの名前変更・削除ができる。テスト予定日も登録できる
- **F-14 データの永続保管**: 登録した問題セットと正誤履歴は、ユーザーが削除するまで端末内に保管(蓄積)する。アプリを起動するとまず過去の問題セット一覧(ホーム画面)が表示され、いつでも見返して練習を再開できる。取り込みは新しいプリントが来たときだけ行う
- **F-15 バックアップ**: 全データ(問題セット・履歴)を JSON ファイルとしてエクスポート/インポートできる。ブラウザのデータ消去や機種変更に備える
- **F-13 API キー設定(親向け設定画面)**: Google Gemini / Anthropic Claude の API キーを設定画面で登録し、端末内(localStorage)にのみ保存する。Gemini は Google AI Studio で無料取得できる(クレジットカード不要)。キーが必要なのは F-1 の写真自動取り込みだけで、未設定でも他の全機能が使える

### 4.2 練習(子ども向け)

- **F-5 練習画面**: スマホ縦画面で、上部に問題文(漢字に直す部分を下線で強調)、下部に手書きエリアを表示する

```
┌─────────────────────┐
│ 6月範囲②   12 / 40  │ ← セット名と進捗
│                     │
│   しきんを集める     │ ← 問題(下線部を強調)
│   ￣￣￣￣           │
├─────────────────────┤
│ ┌─────┐ ┌─────┐    │
│ │     │ │     │    │ ← 手書きマス(文字数分)
│ └─────┘ └─────┘    │
│ [1文字消す] [全部消す]│
│     [答えを見る]     │
└─────────────────────┘
```

- **F-6 手書きエリア**: 指(またはタッチペン)で書ける。正解の文字数分のマス目を表示。マス単位の消去と全消去ができる
- **F-7 自己採点**: 「答えを見る」で正解の漢字を表示し、自分の書いた字と見比べて ○ / × を自分でつける。結果を記録して次の問題へ進む
- **F-8 出題順の切り替え**: 番号順 / ランダム / 間違えた問題だけ(復習モード)
- **F-9 途中中断・再開**: 練習を途中でやめても、続きから再開できる

### 4.3 成績・復習

- **F-10 正誤履歴**: 問題ごとに ○× の履歴を保存する
- **F-11 復習モード**: 直近で×だった問題、正答率の低い問題を優先して出題する
- **F-12 成績画面(親向け)**: セットごとの正答率、苦手な問題(漢字)の一覧を確認できる

### 4.4 将来拡張(v2 以降・今回は作らない)

- **F-20 手書き文字の自動認識・自動採点**: v1 の利用状況を見て追加を検討する
- **F-21 なぞり書きお手本**: 手書きマスに薄く正解を表示してなぞる練習モード
- **F-22 複数ユーザー(兄弟)対応**
- **F-23 問題セットの共有**: セット単位のファイル書き出し/QR コード等で、他の家庭に問題を渡せるようにする(v1 でも F-15 の JSON を送り合えば実質可能)

## 5. 非機能要件

- **N-1 プラットフォーム**: Webアプリ(PWA)。ホーム画面に追加してアプリのように起動できる。iPhone / Android のブラウザで動作し、PC でも閲覧可能
- **N-2 画面設計**: スマホ縦画面を基準にする。手書きエリアは画面下半分を確保し、子どもの指でも書きやすい大きさのマスにする
- **N-3 オフライン動作**: 一度登録した問題セットはオフラインでも練習できる(練習・採点・記録はローカルで完結)。AI取り込みのみオンライン必須
- **N-4 データ保存**: 問題・履歴は端末ローカル(IndexedDB 等)に保存し、削除するまで永続化する。端末をまたぐ同期は v1 では対象外(F-15 のエクスポート/インポートで代替)。ブラウザによるデータ自動削除を避けるため、ホーム画面追加(PWA インストール)を推奨手順とする
- **N-5 操作性**: 子どもが一人で練習を完結できるシンプルな UI。管理機能(登録・成績)は練習動線から分離する
- **N-6 コスト**: AI 取り込み以外はランニングコストをかけない構成にする

## 6. 技術方針(候補)

- ホスティング: GitHub Pages(静的ホスティング・無料)。サーバーは持たない
- フロントエンド: SPA(React 等)+ PWA(Service Worker でオフライン対応)
- 手書き: Canvas によるフリーハンド描画(v1 は描画のみ。認識はしない)
- データ: IndexedDB(問題セット・正誤履歴)、localStorage(API キー・設定)
- AI 取り込み: Claude API をブラウザから直接呼び出す(`anthropic-dangerous-direct-browser-access` ヘッダー使用)。キーは親が設定画面で登録し端末内にのみ保存する(F-13)。コードやリポジトリにキーは含めない
- OCR 貼り付け取り込み(F-16): 外部サービスを使わず、貼り付けられたテキストをアプリ内のパーサーで「番号・問題文・正解」に整形する(完全オフライン・無料)
